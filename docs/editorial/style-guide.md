# The Bitcoin Web Editorial Style Guide

This guide sets the house style for public copy and editorial planning notes.

## Language

- Use British English throughout public copy.
- Use recognise, recognised, recognisable.
- Use organise, organised.
- Use centre.
- Use labour.
- Use behaviour.
- Use licence as a noun and license as a verb.
- Use travelling and travelled.
- Use programme for general programmes and program for computer programs.
- Preserve original spelling in source titles, quotations and proper names.

## Tone

- Serious.
- Calm.
- Historically careful.
- Accessible to a smart beginner.
- Non-promotional.
- Not meme-heavy.
- Not "Bitcoin fixes everything".
- No anti-government slogans.
- Distinguish factual claims, interpretations and analogies.

## Content Layers

### Surface

- 15-35 words.
- Concept first.
- No forced Bitcoin connection.
- Readable in roughly five seconds.

### Deeper

- 3-4 short paragraphs.
- Explain the concept or historical problem.
- Name a trade-off or limitation.
- Introduce Bitcoin mechanisms where relevant.
- State the limits of the analogy.

### Sources

- Use central bibliography records only.
- Do not invent citations.
- Treat source cards as supporting bibliography, not line-level citations.
- Keep statuses as draft or source-needed until human review.

## Graph Language

- Shared nodes may have multiple path depths.
- Use "Path depth", not intrinsic "Depth".
- "Also touches" is informational, not navigational.
